Time-series outputs provide an extensible method for publishing [TenXSummary](https://doc.log10x.com/api/js/#TenXSummary) instances generated by an [aggregator](https://doc.log10x.com/run/aggregate) as metric counters to a variety of time-series DBs (e.g., Prometheus, Datadog).

### :material-robot-happy-outline: Logs to Metrics

TenXSummary metrics provide an effective way of delivering data to ML tools like [CloudWatch](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/CloudWatch_Anomaly_Detection.html) and [Datadog](https://docs.datadoghq.com/monitors/types/anomaly/) to detect surges and prevent over-billing. 

### :octicons-package-dependents-24: Extensions

Output [modules](https://doc.log10x.com/engine/module/) utilize the extensible [Micrometer](https://micrometer.io/) observability framework. To learn more about creating custom metrics outputs, see [metric extensions](https://doc.log10x.com/api/output/#micrometer).
